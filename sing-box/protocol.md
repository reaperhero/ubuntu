一类：真正的代理协议

shadowsocks
轻量、成熟、生态广的代理协议。核心是把流量加密后转发，部署简单，性能通常不错。优点是兼容性强，缺点是不同版本的抗识别能力差异很大，现代场景通常更推荐新一代 AEAD 方案。

trojan
走 TLS 外壳的代理协议，目标是让流量外观看起来更像普通 HTTPS。优点是伪装自然、在很多网络里比较稳；缺点是对证书、域名、TLS 行为更敏感。

vmess
V2Ray 体系里比较老牌的协议，功能很多、历史包袱也多。它曾经非常流行，但这些年整体口碑和推荐度都不如更新的方案。更像一个“功能丰富但不再算最优先选择”的协议。

vless
可以理解为 VMess 的更轻、更干净版本，本身尽量少做事，把认证和传输层解耦。经常和 TLS、XTLS、WebSocket、gRPC 之类组合使用。优点是灵活、现代；缺点是组合多，理解门槛会比单体协议高。

hysteria
基于 UDP / QUIC 思路的一代协议，强调弱网和高丢包环境下的吞吐能力。速度表现可能很好，但流量特征也会更明显。

hysteria2
Hysteria 的新一代版本，设计更简洁，思路更统一。仍然偏重弱网性能和带宽利用率，适合高延迟、丢包环境，但“像普通流量”这件事不如 TLS/TCP 系协议自然。

tuic
也是基于 QUIC/UDP 的现代代理协议，主打低延迟和高性能。它和 Hysteria 2 有相似的场景重叠，但实现风格和细节不同。通常适合追求速度、实时性的人。

naive
NaiveProxy 体系的协议思路，核心是“看起来像普通 HTTPS / HTTP 代理行为”。它的特点不是花哨，而是尽量贴近正常 Web 流量形态，隐蔽性思路比较偏“融入常规互联网”。

anytls
比较新的 TLS 类代理协议，思路上是围绕 TLS 会话来承载代理连接。可以把它理解为“以 TLS 会话为核心的代理形式”，重点是复用 TLS 这一层的外观与连接能力。

shadowtls
这不是独立承载业务流量的大而全协议，更像一个“给别的代理流量再套一层 TLS 外观”的伪装层。常见用途是搭配别的协议一起用，让外观看起来更像正常 TLS 连接。

socks
标准代理协议族，最常见的是 SOCKS5。它不是专门为“翻墙”设计的，更像通用代理接口。优点是简单、很多程序原生支持；缺点是自身不提供高级伪装能力。

http
指 HTTP CONNECT 代理。很多浏览器、系统、企业网络都支持。优点是普适；缺点是能力边界比现代专用代理协议窄，隐蔽性和灵活性也一般。

ssh
利用 SSH 通道来做代理或转发。优点是很多服务器原生就有 SSH，临时使用很方便；缺点是它本来不是专门为大规模代理优化的，性能和伪装性都不是主打。

tor
匿名通信网络，不只是“代理协议”，更像一个分布式匿名路由系统。它的重点是匿名性和多跳转发，不是低延迟或高速度，所以通常会比较慢，但隐私目标和普通代理完全不同。

二类：不是代理协议，而是“直连 / 拦截 / 内部处理”

direct
不是代理协议，就是“直接访问目标”。意思是 sing-box 不再转发给上游代理，而是自己像普通客户端一样去连目标站点。

block
也不是协议，就是“拦截并丢弃”。命中它的流量会被拒绝，常用于广告、恶意域名或某些不想放行的连接。

dns
这是内部 DNS 处理通道，不是拿去连远端代理服务器的协议。它表示请求交给 sing-box 内部的 DNS 逻辑处理。

redirect
这是透明重定向型入站，不是代理协议。它用在 Linux/macOS 这类系统里，把本机或经过本机的流量“截获”到 sing-box 处理，常见于透明代理/旁路由方案。

tun
这是虚拟网卡入站，也不是代理协议。它通过创建一个 TUN 网络接口，把 IP 层流量整体接进 sing-box，所以适合“接管整机流量”或“做网关”。

mixed
这是一个“混合代理入口”，同时支持 SOCKS4/SOCKS4a/SOCKS5 和 HTTP。它的定位是给应用程序提供一个通用本地代理入口，不是新的网络协议。

三类：不是协议，而是“组合器 / 分组器”

group
它不是网络协议，而是把多个出站组合起来统一管理的概念。常见用途是手动切换、自动测速选择、故障切换等。它更像“代理节点管理层”。
四类：隧道 / VPN / 端点类

wireguard
这是 VPN 隧道协议，不是传统意义上的应用层代理。它工作层级更低，像一个加密的三层网络隧道。优点是性能高、实现干净；缺点是流量外观不像 HTTPS 那样自然，更偏“专用隧道”。

tailscale
严格说它也不是 sing-box 自创的代理协议，而是基于 WireGuard 思想的组网系统。重点是设备组网、子网路由、出口节点、身份管理这些能力，适合“多设备互联”而不只是“单纯代理上网”。

怎么快速区分

想要“给应用提供代理端口”：看 mixed、socks、http
想要“接管整机/整段流量”：看 tun、redirect
想要“真正发到远端代理服务器”：看 shadowsocks、trojan、vless、vmess、hysteria2、tuic、naive、anytls
想要“不是代理，直接本机出去”：direct
想要“直接丢掉”：block
想要“做 VPN / 组网”：wireguard、tailscale
想要“节点编排/切换”：group
